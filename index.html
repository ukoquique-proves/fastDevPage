⚖️ Docker en Puppy Linux — Informe Real

Informe de pruebas en TrixiePup64Retro 25.09 · Kernel 6.12.85

Sistema probado: TrixiePup64Retro 25.09 · Kernel 6.12.85 · Intel i7-4600U · 7.8 GB RAM
Docker: 26.1.5+dfsg1 (repos de Debian)

1. Conclusión directa

Docker se instala, arranca y ejecuta contenedores correctamente en esta variante de Puppy Linux. Las afirmaciones de doubted_claim.md están exageradas o incorrectas; a continuación se documenta la refutación y los datos reales.

3. Refutación punto por punto

"Overhead constante"

Falso en la magnitud que implica. El daemon en reposo consume ~137 MB y CPU ~0.1% o menos.

"Requiere 2.5 GB+ de overhead"

Falso para Linux nativo — esa cifra corresponde a Docker Desktop en Windows/macOS. En esta instalación los paquetes ocupan ~248 MB en disco y `/var/lib/docker` ocupa ~520 KB tras instalación limpia.

"Tu laptop secundaria queda inutilizable"

Falso: la máquina de prueba (i7-4600U) funciona con Docker sin problemas; con 2 GB de RAM es posible usar Docker si el kernel y configuraciones son adecuadas.

"Latencia I/O"

Parcialmente cierto: depende del driver de almacenamiento. En esta máquina el driver activo fue vfs (lento). Cambiar a overlay2 (si el rootfs y kernel lo permiten) reduce significativamente la penalización.

Otras notas

Logs y herramientas de diagnóstico funcionan (`docker logs`, `docker inspect`, `docker stats`). `docker-compose` está disponible en los repos (2.26.1).

4. Comandos reproducibles

# Instalación completa
apt-get install -y docker.io docker-cli

# Verificación
docker run --rm hello-world
# → "Hello from Docker!" ✓

# Daemon activo
ps aux | grep dockerd

5. Limitaciones reales

  • Storage driver por defecto: vfs — en Puppy el rootfs puede estar en squashfs/tmpfs, lo que impide usar overlay2 sin pasos adicionales.
  • Sin systemd real — Puppy usa BusyBox init; el daemon arranca pero la gestión por systemd no está disponible por defecto.
  • Warnings en postinstall por `logger` de BusyBox — no impiden el funcionamiento.

6. Veredicto

El documento crítico no refleja la realidad técnica en Linux nativo. Docker 26.1.5 instala, inicia y ejecuta contenedores en Puppy Linux TrixieRetro 25.09 con aproximadamente 137 MB RAM en reposo y ~248 MB de paquetes en disco. La elección entre Docker y flujos nativos debe basarse en hechos y configuración, no en cifras tomadas de entornos distintos como Docker Desktop.
